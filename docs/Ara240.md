## Installation Steps

### Step 1 — Set Hugging Face environment variable

The optional package postinst scripts pull models from Hugging Face. The xet transport
must be disabled before installing them or it will saturate the network.

Write the variable into `/etc/environment` (system-wide, persists across reboots):

```bash
echo 'HF_HUB_DISABLE_XET=1' >> /etc/environment
```

Also create the service EnvironmentFile so the systemd service and proxy daemon inherit it:

```bash
mkdir -p /run/ara240
echo 'HF_HUB_DISABLE_XET=1' > /run/ara240/env
```

Export into the current shell for the remainder of this session:

```bash
export HF_HUB_DISABLE_XET=1
```

### Step 2 — Install the base SDK

```bash
cd ~/kinara_ara240/downloads
dpkg -i rt-sdk-ara2_2.0.4.deb
ldconfig
```

The postinst script automatically:
- Resizes `/dev/mmcblk1p2` to fill available space
- Creates a 5 GB swap file at `/swapfile`
- Creates `/usr/share/cnn` and `/usr/share/llm`
- Installs the `uv` Python package manager
- Enables `rt-sdk-ara2.service` in systemd
- Reloads udev rules

### Step 3 — Patch the systemd service file

The shipped service file has `After=multi-user.target` which creates a dependency loop.
Patch it **before** starting the service:

```bash
cat > /etc/systemd/system/rt-sdk-ara2.service << 'EOF'
[Unit]
Description=Run rt-sdk-ara2 script at boot
After=network.target
Wants=network.target

[Service]
Type=oneshot
ExecStart=/bin/bash /usr/share/rt-sdk-ara240/scripts/rt-sdk-ara2_pre.sh
ExecStart=/bin/bash /usr/share/rt-sdk-ara240/scripts/rt-sdk-ara2_script.sh
ExecStartPost=/bin/bash /usr/share/rt-sdk-ara240/scripts/rt-sdk-ara2_post.sh
EnvironmentFile=-/run/ara240/env
KillMode=control-group
RemainAfterExit=true

StandardOutput=journal+console
StandardError=journal+console

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
```

### Step 4 — Start the service

The service was already enabled by the postinst script; only start it:

```bash
systemctl start rt-sdk-ara2
systemctl status rt-sdk-ara2
```

A healthy start shows three ExecStart/ExecStartPost processes completing with
`status=0/SUCCESS` and a final log line:
```
Hardware bringup is done (1 device(s) configured) and proxy is launched successfully in the background.
```

### Step 5 — Install optional packages

Ensure `HF_HUB_DISABLE_XET=1` is exported (Step 1) before running this.
Model downloads are large and will take significant time on a slow connection.

```bash
cd ~/kinara_ara240/downloads
dpkg -i eiq-aaf-connector_2.0.deb \
        llm-edge-studio_2.0.0.deb \
        vlm-edge-studio_1.0.0.deb \
        ara2-vision-examples_1.0.deb
```

### Step 6 — Update Firmware on Ara240

There is a newer version of firmware available. After running `./chip_info.sh` look for the firmware line. If it is not `2.0.0` then we need to upgrade the firmware. Restart the system and enter `u-boot` by pressing `1` during initial startup.

Add `pcie_aspm=off` to boot args to prevent a stability issue during FW programming. 
``` u-boot
editenv mmcargs
edit: setenv bootargs ${jh_clk} ${mcore_clk} console=${console} root=${mmcroot} pcie_aspm=off
saveenv
boot
```

Make sure that the host PC is connected to the network and run:
```
systemctl stop eiq-aaf-connector
program_flash.sh
```

If successful then `reboot`. You should now see `2.0.0` on the `chip_info.sh`.

## Installing NXP Models

Depending on bandwidth you may need to install the models manually with XET disabled otherwise HF downloads can easily saturate the bandwidth of a router.

``` bash
export HF_TOKEN=****
export HF_HUB_DISABLE_XET=1

uvx --from /usr/share/python-wheels/fetch_models-1.0.0-py3-none-any.whl fetch_models --repo-id nxp/Qwen2.5-7B-Instruct-Ara240

uvx --from /usr/share/python-wheels/fetch_models-1.0.0-py3-none-any.whl fetch_models --repo-id nxp/Qwen2.5-Coder-1.5B-Ara240
```
